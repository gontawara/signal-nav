# ADR-001: 信号機位置データのデータソース選定

- **日付**: 2026-05-13
- **ステータス**: 採用
- **決定者**: gontawara

## 背景

信号最小化ルートナビの実装にあたり、信号機の位置データが必要。日本国内で利用可能なデータソースを調査し、1週間のハッカソンで使用するデータソースを決定する。

## 検討した選択肢

### 1. OpenStreetMap（OSMnx経由）

- **概要**: ボランティアベースの地理情報データベース。`highway=traffic_signals`タグで信号機ノードを取得可能
- **コスト**: 無料
- **カバレッジ**: 推定75%程度（北海道の事例: OSM約9,700件 vs 警察公式データ約13,000件）
- **取得方法**: OSMnxライブラリでPythonから直接取得可能。道路グラフと信号ノードを一括で取得できる
- **制約**: ボランティアベースのため地域差が大きい。更新頻度も一定でない

### 2. 国土地理院「数値地図（国土基本情報）」信号交差点データ

- **概要**: 国土地理院が整備した信号交差点のポイントデータ。信号交差点コード、経緯度、地点名、DRM対応コードを含む
- **コスト**: 有料（2次メッシュ単位で170円）
- **カバレッジ**: 公的機関の整備データのため網羅性が高い
- **取得方法**: Webからの購入手続きが必要。データ形式はGML/Shapefile
- **制約**: 購入手続きに時間がかかる。道路グラフとの紐付けに追加の変換作業が必要

### 3. 都道府県警察の交通安全施設データ

- **概要**: 信号機を実際に管理しているのは各都道府県警察。一部の県では施設計画として総数を公開
- **コスト**: 基本無料（公開されている場合）
- **カバレッジ**: 最も正確（管理元データ）
- **取得方法**: 標準化されたAPIやダウンロード手段がない。公開形態が都道府県ごとに異なる
- **制約**: 福岡県警が位置データをオープンデータとして提供している証拠はなし

### 4. 国土数値情報

- **概要**: 国交省提供の無料GISデータ。道路・交通分野のデータを多数提供
- **コスト**: 無料
- **信号データ**: **信号機の位置データは提供されていない**。道路種別・幅員等の補助データは利用可能
- **制約**: 信号機データがないため、単独では要件を満たさない

### 5. PLATEAU（3D都市モデル）

- **概要**: 国交省の3D都市モデルプロジェクト。福岡市も対象都市
- **信号データ**: 信号機の個別位置データは含まれていない。建物・道路の3Dモデルが主
- **制約**: 信号機データがないため、単独では要件を満たさない

### 6. Digital Road Map（DRM）

- **概要**: カーナビの基盤データ。日本デジタル道路地図協会が提供
- **信号データ**: 含む
- **制約**: 商用ライセンス。一般公開されていない。個人プロジェクトでの利用は現実的でない

## 決定

**OpenStreetMap（OSMnx経由）を採用する。**

## 理由

1. **開発速度**: Pythonから直接取得でき、道路グラフと信号データを一括で扱える。追加の変換作業が不要
2. **コスト**: 無料
3. **1週間の制約**: 国土地理院データは購入手続きとデータ変換に工数がかかり、ハッカソンの期間に見合わない
4. **福岡市での検証結果**: データ検証スクリプトの実行により、プロトタイプとして成立する程度の信号ノードが取得できることを確認済み

## リスクと今後の対応

- **カバレッジの限界**: 推定75%程度のカバレッジであり、一部の信号が欠損している。プロトタイプの制約として明記する
- **実用化に向けた拡張**: プロダクション化する場合は、国土地理院の信号交差点データや警察データとの統合が必要
